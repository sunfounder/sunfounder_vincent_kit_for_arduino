3.5 Sistema di Controllo Accessi
===================================

Panoramica
-----------------

Il sistema di controllo accessi è progettato per gestire i punti di ingresso, 
evolvendosi dal tradizionale lucchetto meccanico. I lucchetti tradizionali, 
pur robusti, possono essere violati o richiedono la sostituzione in caso di 
perdita delle chiavi. Per risolvere questi problemi, sono stati introdotti i 
lucchetti elettronici con schede magnetiche o codici, che hanno migliorato la 
gestione degli accessi, segnando l'inizio dell'era elettronica.

Circuito Fritzing
-----------------------

In questo esempio, il modulo di alimentazione fornisce energia alla breadboard. 
Collega il GND della scheda Mega 2560 alla barra catodica della breadboard, il 
GND di ULN2003 alla barra catodica della breadboard, il VCC all'uscita 5V del 
modulo di alimentazione e il motore passo-passo ai pin OUT1-OUT5 di ULN.

Spiegazione dell'Esempio
-----------------------------

Il flusso di lavoro del sistema di controllo accessi è illustrato nel diagramma di flusso.

Quando non viene identificato un ID autorizzato (la variabile "Approved" 
è uguale a 0), il dispositivo esegue l'identificazione dell'ID e della password.

Se l'ID autorizzato viene identificato ("Approved" è uguale a 1), la porta 
si apre. Successivamente, la porta si chiuderà dopo pochi secondi e lo stato 
di identificazione verrà reimpostato ("Approved" tornerà a 0).

Oltre alla funzione principale di controllo accessi, il progetto utilizza 
un LCD e un buzzer attivo per completare il sistema di interazione con l'utente.
