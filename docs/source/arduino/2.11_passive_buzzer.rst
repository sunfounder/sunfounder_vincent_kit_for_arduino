2.11 Buzzer Passif
=====================

Aperçu
--------

Dans cette leçon, vous allez découvrir le buzzer passif. Ce type de buzzer 
électronique avec une structure intégrée est alimenté par une source de courant 
continu (DC). Le buzzer passif est largement utilisé dans les ordinateurs, 
alarmes, jouets électroniques, téléphones, minuteries, et autres produits 
électroniques ou dispositifs sonores.

Analyse du Code
-------------------

Deux points nécessitent votre attention :

1. ``tone()`` & ``noTone()`` : Cette fonction permet de contrôler directement le son du buzzer passif, dont le prototype est le suivant :

.. code-block:: arduino

    void tone(int pin, unsigned int frequency)
    void tone(int pin, unsigned int frequency, unsigned long duration)

Génère une onde carrée de la fréquence spécifiée (et avec un cycle de travail de 50%) sur une broche (faisant ainsi vibrer le buzzer passif pour produire un son). Une durée peut être spécifiée, sinon l’onde continue jusqu’à un appel de noTone(). La broche peut être connectée à un buzzer piézo ou un autre haut-parleur pour jouer des tonalités.

Une seule tonalité peut être générée à la fois. Si une tonalité est déjà jouée sur une autre broche, l’appel à tone() n’aura aucun effet. Si la tonalité est déjà en cours sur la même broche, l’appel ajustera sa fréquence.

L’utilisation de la fonction ``tone()`` interférera avec la sortie PWM sur les broches 3 et 11 (sauf pour les cartes Mega).

Il est impossible de générer des tonalités inférieures à 31 Hz.

**pin** : La broche de l’Arduino sur laquelle générer la tonalité.

**frequency** : La fréquence de la tonalité en hertz.

**duration** : La durée de la tonalité en millisecondes (facultatif).

.. code-block:: arduino

    void noTone(int pin)

Arrête la génération d'une onde carrée déclenchée par ``tone()``. Sans effet si aucune tonalité n’est générée.

**pin** : La broche de l’Arduino sur laquelle arrêter la tonalité.

Après avoir compris ces deux fonctions, le code devient clair : les tableaux melody[] et noteDurations[] préparent la suite d'appels successifs de la fonction ``tone()`` pour modifier la tonalité et la durée de chaque note jouée dans la boucle afin d'obtenir un effet musical.

2. ``pitches.h`` : Le code utilise un fichier supplémentaire, ``pitches.h``. Ce fichier contient toutes les valeurs de fréquence pour les notes musicales standards. Par exemple, NOTE_C4 est le do médian, NOTE_FS4 est un fa dièse, etc. Cette table de notes a été écrite par Brett Hagman, sur laquelle la commande ``tone()`` est basée. Elle est utile chaque fois que vous souhaitez jouer des notes musicales.

.. code-block:: arduino

    #include "pitches.h"

.. note::
    Un fichier ``pitches.h`` est déjà inclus dans cet exemple. Si vous le placez dans le même dossier que le code principal, les étapes d'installation de ``pitches.h`` peuvent être ignorées.
